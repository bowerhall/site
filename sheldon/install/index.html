Install Sheldon

Two paths: quick install for personal use, or CI/CD for automated deployments. Both get you the same Sheldon.

Quick Install

Best for: personal VPS, homelab, Raspberry Pi. One command, edit config, done.

Run installer
SSH into your server and run. It'll ask for your tokens, configure everything, and start Sheldon.
curl -fsSL bowerhall.ai/sheldon/install | bash
  • 1. Get a server Any Linux machine with 2GB+ RAM. Hetzner, DigitalOcean, Linode, or your own hardware. ~$5-8/mo for a basic VPS.
  • 2. Get a Telegram bot Message @BotFather on Telegram. Send /newbot, follow prompts, save the token.
  • 3. Get your chat ID Message @userinfobot on Telegram. It replies with your numeric ID.
  • 4. Get an LLM API key Kimi (recommended, cheapest), Anthropic, or OpenAI. One is enough — Sheldon auto-detects.
CI/CD Install

Best for: developers who want automated deployments on every push. Fork, configure secrets, push.

  • 1. Fork the repo Fork bowerhall/sheldon to your GitHub account.
  • 2. Get a server Same as quick install. Note the IP address and set up SSH key access.
  • 3. Configure secrets In your fork: Settings → Secrets → Actions. Add these repository secrets:
Required secrets
Add these in GitHub repo settings:
# Server access VPS_HOST=your-server-ip VPS_USER=root VPS_SSH_KEY=-----BEGIN OPENSSH PRIVATE KEY-----... # Container registry GHCR_TOKEN=ghp_xxxx # GitHub PAT with packages:write # Bot & LLM TELEGRAM_TOKEN=your-bot-token OWNER_CHAT_ID=your-chat-id KIMI_API_KEY=your-kimi-key TZ=Europe/London # your timezone # Domain (for HTTPS) DOMAIN=yourdomain.com ACME_EMAIL=you@email.com
  • 4. Push to deploy Push to main branch. GitHub Actions builds, pushes to GHCR, and deploys to your server automatically.
  • 5. Auto-updates Watchtower runs on the server and pulls new images every 5 hours. Or trigger manually.
Managing Sheldon
Manage config
Change API keys, timezone, or other settings and restart Sheldon:
curl -fsSL bowerhall.ai/sheldon/config | sudo bash
Add a domain
Upgrade from IP-only to HTTPS with your own domain name:
curl -fsSL bowerhall.ai/sheldon/upgrade-domain | sudo bash
Troubleshooting
  • Sheldon not responding Check logs for errors. Usually an invalid API key or rate limit. Sheldon tries kimi → claude → openai in order.
  • API errors / rate limits Add credits to your provider, or add another API key. Sheldon auto-falls back between configured providers.
  • Docker issues Run docker compose down && docker compose up -d to restart everything cleanly.
  • First startup slow Ollama downloads embedding model (~274MB) on first container start. First message after restart may also be slow.
