sheldonmem

Memory backend for AI agents. One SQLite file stores entities, facts, relationships, and vectors. No Postgres, no Redis, no Pinecone. Copy one file to back up everything.

Install
go get github.com/bowerhall/sheldon/pkg/sheldonmem
Data model
  • Entities People, places, projects, concepts. Named nodes in the graph.
  • Facts Key-value pairs attached to entities. Confidence-scored. Domain-tagged.
  • Edges Typed relationships between entities. "works_at", "lives_in", "knows".
  • Vectors sqlite-vec for semantic search. Bring your own embedder.
Features
  • 14 life domains Identity, health, relationships, work, finances, goals, routines, etc. Facts tagged for scoped retrieval.
  • Hybrid search Keyword matching + vector similarity in one query. Results ranked by combined score.
  • Contradiction resolution New facts supersede old ones on the same field. "city=NYC" â†’ "city=Portland" auto-archives NYC.
  • Time-based queries "since: 2025-02-20", "time_range: last_week". Explicit temporal retrieval.
  • Salience decay Old, unused facts fade. Keeps agent context relevant over time.
Usage
store, _ := sheldonmem.Open("memory.db") store.SetEmbedder(embedder) // Store facts store.AddFact(&userID, domainWork, "role", "engineer", 0.9) store.AddFact(&userID, domainLocation, "city", "Portland", 0.9) // Hybrid recall results, _ := store.Recall(ctx, "where does user work", nil, 5) // Time-based recall results, _ := store.Recall(ctx, "*", &RecallOpts{ TimeRange: "last_week", }, 10)
